# MEO勝ち上げくん：自動化できること・できないことの整理

## 前提：Googleが許していること・嫌うこと

Googleは「公式API経由の操作」は許可しているが、「スクレイピング（画面を機械的に読み取る行為）」は禁止している。この違いが全ての判断基準になる。

---

## 自動化できること（安全）

### 1. GBP投稿の「文章生成」
- **やっていること**: AIが投稿文を作成する → ユーザーが確認 → ユーザー自身が手動でGBPに投稿
- **なぜ安全か**: Googleに触れていない。文章を作っているだけ。Word で文章を書くのと同じ
- **リスク**: なし

### 2. FAQ・ブログ記事の生成
- **やっていること**: AIがFAQやブログ記事を作成する
- **なぜ安全か**: 自分のサイトに載せるコンテンツを作っているだけ。Googleは関係ない
- **リスク**: なし

### 3. 口コミ返信文の生成
- **やっていること**: AIが口コミへの返信文を作成する → ユーザーが確認 → ユーザー自身が手動で返信
- **なぜ安全か**: 文章を作っているだけ。実際の返信操作はユーザーが行う
- **リスク**: なし

### 4. note・WordPress への自動投稿
- **やっていること**: アプリからnoteやWordPressのAPIを使って記事を投稿
- **なぜ安全か**: note・WordPressは公式にAPIを公開しており、自動投稿を許可している
- **リスク**: なし

### 5. MEO順位チェック（SerpAPI経由）※注意あり
- **やっていること**: SerpAPIを使ってGoogle検索結果から順位を取得
- **注意点**: SerpAPIは2025年12月にGoogleから訴訟を起こされている（係争中）。現時点では使えるが、将来的にサービスが停止する可能性がある
- **リスク**: 中（サービス側のリスク。ユーザーのGBPアカウントには影響しない）

---

## 自動化できないこと（危険 or 不可能）

### 1. GBP投稿の「自動投稿」（APIから直接Googleに投稿）
- **なぜできないか**: Google Business Profile APIは2024年に大幅制限され、投稿機能へのアクセスは審査制。一般的なサードパーティアプリが自由に投稿APIを使うことはできない
- **代替策**: 文章はAIが生成 → ユーザーがGBPの管理画面からコピペで投稿（1分で終わる）
- **やったらどうなるか**: 非公式な方法（スクレイピング等）で自動投稿すると、GBPアカウントの停止リスクがある

### 2. 口コミの自動取得（Googleマップからスクレイピング）
- **なぜできないか**: Googleの利用規約でスクレイピングは明確に禁止。SerpAPIもGoogleから訴訟を起こされている
- **代替策**: ユーザーが口コミをコピペで設定画面に貼る（代表的な5件程度で十分）
- **やったらどうなるか**: 直接的にGBPに悪影響はないが、スクレイピングサービスが突然使えなくなるリスクがある

### 3. 口コミの自動返信（APIから直接Googleに返信）
- **なぜできないか**: GBP APIの口コミ返信機能も審査制。未審査のアプリからは使えない
- **代替策**: 返信文はAIが生成 → ユーザーがGBPの管理画面から手動で返信
- **やったらどうなるか**: 非公式な方法は規約違反

### 4. 写真の自動アップロード
- **なぜできないか**: GBP APIの写真機能も制限されている
- **代替策**: ユーザーがGBP管理画面から手動アップロード

## まとめ：MEO勝ち上げくんのスタンス

| 機能 | 自動化 | やり方 |
|---|---|---|
| GBP投稿の文章作成 | OK | AIが生成、ユーザーが手動投稿 |
| FAQ・ブログ記事作成 | OK | AIが生成 |
| 口コミ返信文の作成 | OK | AIが生成、ユーザーが手動返信 |
| note投稿 | OK | API経由で自動投稿 |
| WordPress投稿 | OK | API経由で自動投稿 |
| GBP投稿の自動投稿 | NG | Googleが許可していない |
| 口コミの自動取得 | NG | スクレイピング禁止 |
| 口コミの自動返信 | NG | Googleが許可していない |
| MEO順位チェック | 注意 | SerpAPI経由（訴訟リスクあり） |

### 一言で説明するなら

> 「AIが文章を作るところまでは全自動。Googleへの操作はユーザー自身がやる。これがGoogleに嫌われない唯一の方法です。」

### なぜこの方針なのか

Googleは「自社のプラットフォームを機械的に操作されること」を嫌う。理由は：
1. 品質管理ができなくなる（スパム投稿が増える）
2. 広告収益に影響する（自動化されると広告を見てもらえない）
3. ユーザー体験が下がる（低品質なコンテンツが増える）

逆に「良質なコンテンツを人間が投稿すること」はGoogleが最も歓迎する行為。MEO勝ち上げくんは「良質なコンテンツを作る」部分をAIで効率化している。投稿は人間がやるので、Googleのポリシーに完全に準拠している。

## SerpAPIについての補足

- 2025年12月にGoogleがSerpAPIを提訴（スクレイピングによる規約違反）
- 2026年2月にSerpAPIが棄却申立て（「公開情報へのアクセスは合法」と主張）
- 2026年5月19日に裁判所の判断予定
- 現時点ではサービスは稼働中だが、判決次第で停止の可能性あり
- MEO順位チェック機能は、SerpAPIが使えなくなった場合の代替手段（Google公式のSearch Console連携）も既に実装済み
